介面介紹

AsstAppendTask

介面原型

AsstTaskId ASSTAPI AsstAppendTask(AsstHandle handle, const char* type, const char* params);

介面說明

新增任務。

回傳值

  • AsstTaskId
    若新增成功，回傳該任務 ID，可用於後續設定任務參數；
    若新增失敗，回傳 0。

參數說明

handleRequiredAsstHandle

執行個體控制代碼 (Handle)

typeRequiredconst char*

任務類型

paramsRequiredconst char*

任務參數，json String

任務類型一覽
  • StartUp
    開始喚醒

enableOptionalboolean

true

是否啟用本任務。

client_typeRequiredstring

用戶端版本。

選項：Official | Bilibili | txwy | YoStarEN | YoStarJP | YoStarKR

start_game_enabledOptionalboolean

false

是否自動啟動用戶端。

account_nameOptionalstring

切換帳號，預設不切換。

僅支援切換至已登入的帳號。依據登入名稱進行搜尋，請確保輸入內容在所有帳號中具有唯一性。

官服：123****4567，可輸入 123****4567、4567、123、3****4567

B 服：張三，可輸入 張三、張、三

Example
{
   "enable": true,
   "client_type": "Official",
   "start_game_enabled": true,
   "account_name": "123****4567"
}

  • Fight
    理智作戰

enableOptionalboolean

true

是否啟用本任務。

stageOptionalstring

關卡名稱。預設為空，將辨識 目前/上次 關卡。不支援在執行中更改設定。
目前支援導航的關卡包含：

  • 全部主線關卡：可在關卡末尾新增 -NORMAL 或 -HARD 以切換標準或磨難難度。
  • 資源收集：
    • 龍門幣、作戰記錄： 支援第 5 / 6 關，但必須輸入 CE-6 / LS-6。若第六關無法代理，MAA 會自動切換至第五關。
    • 技能書、採購憑證、碳本：支援第 5 關，必須輸入 CA-5 / AP-5 / SK-5。
    • 晶片關卡：支援所有關卡，必須輸入完整編號，例如PR-A-1。
  • 剿滅作戰：支援以下傳入值（必須使用對應的 Value）：
    • 當期剿滅：Annihilation
    • 切爾諾伯格：Chernobog@Annihilation
    • 龍門外環：LungmenOutskirts@Annihilation
    • 龍門市區：LungmenDowntown@Annihilation
  • 別傳：支援 OF-1 / OF-F3 / GT-5。
  • 當期 SideStory 活動：支援活動後三關。可查閱 API 獲取支援的關卡列表。需額外加載 tasks.json 中的活動關卡導航。
  • 復刻 SideStory 活動：輸入 SSReopen-<關卡前綴> ，可一次性刷完 XX-1 ~ XX-9 關，如 SSReopen-IC。

medicineOptionalnumber

0

理智藥最大使用量。

expiring_medicineOptionalnumber

0

48 小時內過期理智藥最大使用量。

stoneOptionalnumber

0

碎石最大數量。

timesOptionalnumber

2147483647

戰鬥次數。

seriesOptionalnumber

連戰次數，-1 ~ 6。
-1：停用自動切換連戰。
0：自動切換為當前可用的最大連戰次數（若理智不足以進行 6 連戰，則選擇最低可用次數）。
1 ~ 6：指定固定的連戰次數。

dropsOptionalobject

指定物品掉落數量，預設不指定。key 為 item_id，value 為數量。key 可參考 resource/item_index.json 檔案。

例如：{ "30011": 10, "30062": 5 }

上述所有條件（次數、掉落等）均為「或」的關係，任一條件達成即停止任務。

report_to_penguinOptionalboolean

是否回報企鵝物流

penguin_idOptionalstring

企鵝物流回報 ID，預設為空。僅在 report_to_penguin 為 true 時有效。

report_to_yituliuOptionalboolean

是否回報一圖流。

yituliu_idOptionalstring

一圖流回報 ID，預設為空。僅在 report_to_yituliu 為 true 時有效。

serverOptionalstring

選項：CN | US | JP | KR

client_typeOptionalstring

用戶端版本，預設為空。用於遊戲崩潰後自動重啟並接續任務；若為空則不啟用自動重啟功能。
選項：Official | Bilibili | txwy | YoStarEN | YoStarJP | YoStarKR

DrGrandetOptionalboolean

在碎石確認介面等待，直到目前理智自動恢復 1 點後，再立刻執行碎石回復，以避免浪費自然恢復。

  • Recruit

true

是否啟用本任務。

refreshOptionalboolean

false

是否刷新三星 Tags。

selectRequiredarray<number>

會被自動點選的 Tag 等級。

confirmRequiredarray<number>

會被自動點選確認的 Tag 等級。若只想計算結果而不實際招募，請設定為空陣列。

first_tagsOptionalarray<string>

首選 Tags，僅在 Tag 等級為 3 時有效，預設為空。
當 Tag 等級為 3 時，會盡可能多選擇此處設定的 Tags（若有出現），且為強制選擇，會忽略所有 "讓 3 星 Tag 不被選擇" 的設定。

extra_tags_modeOptionalnumber

0

選擇更多的 Tags。
0：預設行為。
1：選滿 3 個 Tags，即使可能產生衝突。
2：儘可能同時選擇更多高星標籤組合，即使可能產生衝突。

timesOptionalnumber

0

招募總次數。若僅需公招計算，請設定為 0。

set_timeOptionalboolean

true

是否設定招募時限。僅在 times 為 0 時才套用。

expediteOptionalboolean

false

是否使用加急許可。

expedite_timesOptionalnumber

加急次數，僅在 expedite 為 true 時有效。預設無限次使用（直到 times 達到上限）。

skip_robotOptionalboolean

true

是否在辨識到小車 Tag 時跳過。

recruitment_timeOptionalobject

Tag 等級（大於等於 3）對應的期望招募時限（單位：分鐘）。預設皆為 540 分鐘（即 09:00:00）。
範例：{ "3": 540, "4": 540 }

report_to_penguinOptionalboolean

false

是否回報企鵝物流。

penguin_idOptionalstring

企鵝物流回報 ID，預設為空。僅在 report_to_penguin 為 true 時有效。

report_to_yituliuOptionalboolean

是否回報一圖流數據。

yituliu_idOptionalstring

一圖流回報 ID，預設為空。僅在 report_to_yituliu 為 true 時有效。

serverOptionalstring

CN

遊戲伺服器，會影響數據上傳。
選項：CN | US | JP | KR

  • Infrast
    基建換班

enableOptionalboolean

true

是否啟用本任務。

modeOptionalnumber

0

換班工作模式。
0 - Default：預設換班模式，單設施最優解。

  • Mall
    領取信用點數及商店購物。
    會先依據 buy_first 清單依序購買；接著由左至右掃描商店，避開 blacklist 進行第二輪購買；若信用點數溢出，則會無視黑名單再次由左至右購買，直到信用點數不再溢出為止。

enableOptionalboolean

true

是否啟用本任務。

visit_friendsOptionalboolean

true

是否訪問好友基建以獲得信用點數。

shoppingOptionalboolean

true

是否執行購物流程。

buy_firstOptionalarray<string>

[]

優先購買清單。填入商品名稱，例如 "招聘許可"、"龍門幣" 等。

blacklistOptionalarray<string>

[]

購物黑名單。填入商品名稱，例如 "加急許可"、"傢俱零件" 等。

force_shopping_if_credit_fullOptionalboolean

false

當信用點數溢出時，是否無視黑名單強制購買。

only_buy_discountOptionalboolean

false

是否僅購買折扣商品，此設定僅對第二輪購買有效。

false

credit_fightOptionalboolean

false

是否借助戰打一次 OF-1 關卡，以便在隔日獲取更多信用點數。

formation_indexOptionalnumber

打 OF-1 關卡時所使用的編隊欄位編號。
範圍為 0–4 的整數，其中 0 表示選擇目前編隊，1-4 分別代表第一、二、三、四編隊。

   "only_buy_discount": true,
   "reserve_max_credit": false,
   "credit_fight": false,
   "formation_index": 0
}

  • Roguelike

enableOptionalboolean

true

Phantom

主題。
Phantom - 傀影與猩紅孤鑽
Mizuki - 水月與深藍之樹
Sami - 探索者的銀淞止境
Sarkaz - 薩卡茲的無終奇語
JieGarden - 歲的界園誌異

modeOptionalnumber

0

模式。
0 - 刷分 / 獎勵點數：盡可能穩定地打更多層數。
1 - 刷源石錠：第一層投資完後立即退出。
2 - 已棄用 兼顧模式 0 與 1，投資後再退出；若無投資則繼續往後打。

4 - 凹開局：先在難度 0 下到達第三層後重啟，再到指定難度下凹開局獎勵。若未獲得「熱水壺」或「希望」則回到難度 0 重新開始。在 Phantom 主題下則不切換難度，僅在目前難度下嘗試到達第三層、重開、凹開局。

6 - 刷月度小隊獎勵：除了模式適配外，其餘邏輯同模式 0。
7 - 刷深入調查獎勵：除了模式適配外，其餘邏輯同模式 0。
