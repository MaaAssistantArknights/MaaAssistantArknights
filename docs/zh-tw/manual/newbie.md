# 新手上路

一句話教學！

## 前置準備

:::: steps

1. 確認系統版本

   MAA 在 Windows 下僅支援 10 和 11，舊版 Windows 用戶請參閱 [常見問題](./faq.md#系統問題) 中的系統問題部分。

   非 Windows 用戶請參閱 [模擬器及設備支援](./device/)。

2. 下載正確的版本

   [MAA 官網](https://maa.plus/) 一般會自動選擇正確的版本架構，對於大多數閱讀本文的用戶來說，應為 Windows x64。

3. 正確解壓縮

   確認解壓縮完整，並確保將 MAA 解壓縮到一個獨立的資料夾中。  
   請勿將 MAA 解壓縮到如 `C:\`、`C:\Program Files\` 等需要系統管理員權限的路徑。

4. 安裝執行庫

   MAA 需要 VCRedist x64 和 .NET 10，請執行 MAA 目錄下的 `DependencySetup_依赖库安装.bat` 進行安裝。

   更多資訊請參閱 [常見問題](./faq.md) 置頂內容。

5. 確認模擬器支援

   查看 [模擬器和設備支援](./device/)，確認正在使用的模擬器支援情況。

6. 正確設定模擬器解析度

   模擬器解析度應為橫向顯示的 `1280x720` 或 `1920x1080`；對於美服（YoStar EN）玩家，必須設定為 `1920x1080`。
   ::::

## 初始配置

0. 若需要使用自動偵測，請僅執行**一個**模擬器，並確保沒有其他安卓設備連接電腦。

1. 跟隨設定指引進行配置，MAA 會自動偵測正在執行的模擬器。若偵測失敗、需要多開模擬器等情況，請參閱 [連線設定](./connection.md)。

2. ~~第一次執行 MAA 時會進行熱更新，請在右側日誌處提示重啟後，關閉 MAA 並重新啟動。~~

3. 拖曳左側任務列表對任務進行排序，勾選或取消勾選核取方塊來選擇要執行的任務。

4. Link Start!

1. 執行日誌位於 MAA 資料夾下的 `debug` 資料夾內。  
日誌很重要，非常重要。向他人尋求協助時，請務必帶著 `asst.log` 和 `gui.log` 一同前往。

2. MAA 為了盡可能滿足所有用戶的各種需求，提供了大量的自定義選項。  
當您認為 MAA 缺少某項功能時，有可能只是沒找到而已，例如 `手動輸入關卡名` 和 `不將已進駐的幹員放入宿舍`。

3. 將滑鼠指標懸停在某些選項上會顯示詳細說明，例如 `不將已進駐的幹員放入宿舍`。

4. 對於部分核取方塊來說，在其上方點擊滑鼠右鍵可以將其變為半選狀態。  
這種狀態下的方塊會在 MAA 下一次啟動時自動清除，可以理解為 `僅執行一次`。
