# 基建換班

## 一般模式

### 換班策略

- 自動計算並選擇**單一設施內的最佳解**，支援所有通用類技能和特殊技能組合。
- 自動辨識作戰紀錄、赤金、源石碎片、晶片，分別使用相應的幹員組合。
- 自動按照 `無人機用途` 選擇的方式使用無人機。
- 自動辨識心情進度條，將剩餘心情百分比小於 `基建工作心情門檻值` 的幹員進駐宿舍。

### 額外說明

- 基建換班目前均為單一設施最佳解，而非跨設施的全域最佳解。
  - 可辨識並使用的例子：`巫戀 + 龍舌蘭`、`紅雲 + 稀音`。
  - 不可辨識並使用的例子：`迷迭香體系`、`紅松騎士團`。
- 若 `無人機用途` 選擇 `貿易站 - 龍門幣`，則會額外辨識 `巫戀組` 並優先為其使用。
- 會客室僅缺一個線索時，會選擇對應線索傾向的幹員，其餘情況則會選擇通用幹員。
- 會客室每次換班都會嘗試贈送線索。官服 (Official, BiliBili) 在 2025-12-05 新增了「一鍵贈送重複線索」功能：當偵測到該按鈕時，MAA 會自動點選以批次贈送重複線索；EN/JP/KR 預計約六個月後會支援該功能，TW 預計約一年後支援（由於台服正在趕進度可能會更早支援）。
- 開啟 `不將已進駐的幹員放入宿舍` 設定項可避免 `艾麗妮`、`邏各斯` 等幹員在訓練室未訓練時被進駐到宿舍，但同時也不會將加工站中心情不滿的幹員進駐宿舍。
- 控制中樞策略太過複雜，目前只考慮 `阿米婭`、`詩懷雅`、`凱爾希`、`彩虹小隊` 及其他心情 +0.05 的幹員，後續將逐步優化。
- 可自行選擇需要 MAA 處理的設施類別，預設全選。

## 隊列輪換

- 該模式需要先在遊戲內設定好預設隊列，MAA 會自動進行輪換。

## 自定義基建換班

- 一圖流的大佬們幫忙寫了一個 [排班產生器](https://ark.yituliu.cn/tools/schedule)，可參閱 [基建協定文件](../../protocol/base-scheduling-schema.md) 使用。
- MAA 資料夾下 `/resource/custom_infrast/` 中內建了幾套理論極限效率的作業，可用作參考。由於其對幹員及練度的需求極高，不推薦直接使用。
